# Инструкция: подключение трекинга (Google Ads + GA4)

Документ для разработчика. Цель — чтобы Google Ads видел реальные брони, а не только посещения.

## Текущее состояние (проверено по коду)

- ✅ Базовый тег Google Ads `AW-18092092950` стоит в `index.html` (строки 41–47). **Повторно добавлять не нужно** — будет двойной счёт.
- ❌ Событие конверсии нигде не вызывается — Google не считает брони.
- ❌ GA4 (`G-…`) и GTM (`GTM-…`) отсутствуют.
- ❌ `gclid` не ловится и не сохраняется с заявкой.

Нужно сделать 3 вещи ниже. Для каждой нужен ID, который выдаётся в кабинете (отмечено ⚠️).

---

## 1. GA4 — аналитика посещений

⚠️ Нужен **Measurement ID** вида `G-XXXXXXXXXX`: создать ресурс в Google Analytics (analytics.google.com → Администратор → Создать ресурс → веб-поток данных для `www.unicar-rent.com`).

Библиотека `gtag.js` уже загружена, поэтому достаточно добавить **одну строку** в `index.html` сразу после строки `gtag('config', 'AW-18092092950');`:

```html
<script>
  gtag('config', 'G-XXXXXXXXXX'); <!-- вставить свой Measurement ID -->
</script>
```

---

## 2. Конверсия Google Ads — отправка брони

⚠️ Нужен **ярлык конверсии** вида `AW-18092092950/AbCdEfGh123`: в Google Ads → Цели → Конверсии → «+ Новое действие» → «Сайт» → действие типа «Отправка формы для потенциальных клиентов». Google выдаст `send_to` (ID аккаунта + ярлык).

Конверсию вешаем на страницу подтверждения брони — она открывается с кодом `?ref=...` только после успешной заявки.

Файл: `src/pages/BookingConfirmationPage.tsx`. Добавить `useEffect` (импортировать `useEffect` из React):

```tsx
useEffect(() => {
  if (referenceCode && typeof (window as any).gtag === 'function') {
    (window as any).gtag('event', 'conversion', {
      send_to: 'AW-18092092950/ВАШ_ЯРЛЫК', // вставить ярлык из Google Ads
      transaction_id: referenceCode,       // защита от двойного засчёта
    });
  }
}, [referenceCode]);
```

После этого каждая успешная бронь = 1 конверсия в Google Ads. `transaction_id` не даст посчитать одну бронь дважды при перезагрузке страницы.

## 3. Офлайн-конверсии (gclid) — опционально, точная оптимизация

Чтобы оптимизироваться не на «отправил форму», а на «бронь реально подтвердили/оплатили».

**3.1. Ловить gclid при заходе с рекламы.** Добавить при старте приложения (например в `App.tsx` или отдельный хук):

```ts
const gclid = new URLSearchParams(window.location.search).get('gclid');
if (gclid) localStorage.setItem('gclid', gclid);
```

**3.2. Отправлять gclid с заявкой.** В `src/api/carsApi.ts` в `createBooking` добавить в payload:

```ts
gclid: localStorage.getItem('gclid') || undefined,
```

**3.3. Бэкенд (UNICAR-Table).** Сохранять `gclid` в записи брони. Когда менеджер ставит статус «подтверждена/оплачена» — выгружать в Google Ads строку: `gclid + конверсия + сумма` (вручную CSV: Инструменты → Конверсии → Загрузки; или через Google Ads API).

## Как проверить, что работает

- Расширение **Google Tag Assistant** (tagassistant.google.com) → открыть сайт → увидеть срабатывание `AW-18092092950` и `G-…`.
- Сделать тестовую бронь → на странице подтверждения Tag Assistant должен показать событие `conversion`.
- В Google Ads конверсия появится со статусом «Записывает» в течение нескольких часов.
